# Портреты Авроры для страницы /audit

Положите сюда 4 файла со строго такими именами (код грузит их по этим путям):

```
aurora-greeting.png    — улыбается (приветствие на /audit и заголовок результата)
aurora-thinking.png    — думает / печатает (во время анализа, в typing-bubble)
aurora-happy.png       — радуется (оценка ≥ 7)
aurora-empathy.png     — сочувствует (оценка < 7)
```

## Требования

- Формат: **PNG с прозрачным фоном** (или WebP — тогда замените расширение в `audit.js` константе `AURORA_PORTRAITS`).
- Размер: **рекомендуем 512×512** (с запасом под Retina; в UI выводятся 96×96 / 40×40).
- Стиль: единый — одна и та же девушка, узнаваемая, дружелюбная, современная.

## Как работает fallback

Если файла на сервере нет, `<img>` сам себя скрывает через `onerror` — layout не ломается,
просто портрета не будет. Поэтому код задеплоен заранее, а вы можете залить картинки позже.

## Как переключаются портреты

Логика в `aurora-landing/audit.js`:

| Экран                | Портрет          |
|----------------------|------------------|
| /audit, форма        | `aurora-greeting`|
| Loading screen       | `aurora-thinking`|
| Результат, score ≥ 7 | `aurora-happy`   |
| Результат, score < 7 | `aurora-empathy` |
